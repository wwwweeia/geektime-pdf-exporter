# 极客时间课程 PDF 批量导出工具 — 概要设计

## 一、项目背景与目标

极客时间作为技术付费内容平台，用户有离线阅读、归档备份的需求。平台本身不提供 PDF 导出功能，且页面为 Vue SPA，内容通过 AJAX 动态加载，传统爬虫方案无法直接获取正文。

**目标**：在已购课程的前提下，将课程全部文章批量导出为干净、可读的 PDF 文件（仅保留正文，去除侧边栏、导航栏等干扰元素）。

---

## 二、技术选型

| 维度 | 选型 | 原因 |
|------|------|------|
| 浏览器自动化 | Playwright (playwright-core) | 原生支持 SPA 渲染等待、网络拦截、PDF 生成，API 比 Puppeteer 更现代 |
| 浏览器引擎 | 本地 Chrome（非 Chromium） | 极客时间检测 Headless Chromium 的 UA，返回 451；必须使用真实 Chrome |
| 运行时 | Node.js (ESM) | Playwright 生态首选，单文件脚本足够 |
| 登录态持久化 | Playwright `storageState` | 保存 cookies + localStorage 到 JSON 文件，复用登录态避免重复登录 |

**不选 HTTP 爬虫的原因**：极客时间是重度 SPA，正文内容通过 AJAX 动态渲染，且有反爬策略（UA 检测、cookie 过期机制），直接 HTTP 请求拿不到渲染后的正文。

---

## 三、系统架构

单文件架构（`export-pdf.mjs`），约 400 行，无额外框架依赖。

```
┌─────────────────────────────────────────────────────────┐
│                       CLI 入口层                         │
│  parseArgs() → 单篇模式 / 批量模式                        │
└──────────────┬──────────────────────┬───────────────────┘
               │                      │
        单篇模式 singleExport    批量模式 batchExport
               │                      │
               ▼                      ▼
┌──────────────────────────────────────────────────────────┐
│                     登录管理层                            │
│  loginAndSaveState()  → 有界面登录，保存 storageState      │
│  loadAuthState()      → 加载并过滤过期 cookie             │
│  checkLoginState()    → 验证登录态是否有效                 │
└──────────────────────┬───────────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────────┐
│                     文章采集层                            │
│  scrapeArticleUrls() → 调用极客时间 API 获取课程文章列表    │
└──────────────────────┬───────────────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────────────┐
│                     PDF 导出层                            │
│  doExport()           → 预扫描 + 3 并发 worker 调度       │
│  exportArticlePdf()   → 单篇：渲染 → 提取 → 替换 → PDF    │
│    ├─ waitForFunction  等待正文容器渲染完成                 │

│    └─ page.pdf()        导出 A4 PDF + 体积校验            │
└──────────────────────────────────────────────────────────┘
```

## 四、核心流程设计

### 4.1 登录态管理

```
首次运行 → 有界面浏览器 → 用户手动/自动登录 → storageState 持久化
后续运行 → 加载 storageState → 检查有效性 → 过期则重新登录
```

**设计要点**：
- 首次运行使用有头模式（`headless: false`），弹出浏览器窗口让用户登录
- 登录成功后通过 `context.storageState()` 将 cookies + localStorage 序列化到 `.auth/storage-state.json`
- 后续运行加载该文件，使用无头模式复用登录态
- 启动时主动访问首页检测登录态（判断页面是否有"登录"按钮），过期则重新触发有头登录

### 4.2 文章列表获取

极客时间的课程目录是 Vue 组件渲染的 `<div>`，没有 `<a>` 标签指向文章 URL，DOM 中无法直接抓取链接。

**解决方案**：在已登录的页面上下文中直接调用极客时间的内部 API：

```
POST https://time.geekbang.org/serv/v1/column/articles
Body: { cid: 课程ID, size: 500, prev: 0, order: 'earliest', sample: false }
Response: { code: 0, data: { list: [{ id, article_title }, ...] } }
```

课程 ID 从用户输入的课程 URL 中正则提取（如 `/column/intro/100066601` → `100066601`）。

**为什么在页面上下文中调 API**：直接 fetch 调用需要携带完整的 cookie（包含 `acw_tc` 等动态 token），在 Playwright 的 `page.evaluate` 中发起请求可以自动携带当前域的所有 cookie。

### 4.3 正文提取与 PDF 生成

这是整个工具的核心难点。极客时间文章页的 DOM 结构特点：
- 页面是 Vue SPA，正文内容通过 AJAX 异步加载，初始 HTML 中没有正文
- 正文中包含懒加载的图片，需要滚动触发加载
- 页面包含侧边栏、导航栏、评论区等大量干扰元素

**提取策略（四步走）**：

```
Step 1: 等待渲染  →  waitForFunction 轮询正文容器内文本 > 200 字
Step 2: 触发懒加载 →  自动滚动到底部再回到顶部，触发所有图片加载
Step 3: 替换页面  →  定位正文容器，document.body.innerHTML = 正文HTML
Step 4: PDF 导出  →  添加排版样式后调用 page.pdf() 生成 A4 格式 PDF
```

正文容器的定位通过一组候选选择器按优先级尝试：
```
[class*="mainAreaWrapper"] → [class*="article-detail"] → [class*="article-content"] → article → [class*="content"]
```

### 4.4 批量导出与并发控制

```
预扫描输出目录 → 文件名前缀匹配已有 PDF → 跳过已导出文章
3 并发 worker → 共享任务队列（原子递增索引）→ 每篇最多重试 1 次
PDF < 50KB 视为空白 → 删除后自动重试
```

**断点续导**：通过文件名前缀匹配实现。导出文件名取文章标题（清理非法字符后），启动时先扫描输出目录的 PDF 文件名，与待导出文章标题前缀比对，已存在的直接跳过。

## 五、遇到的问题与解决方案

### 5.1 反爬检测 — Headless UA 返回 451

**现象**：使用默认 Headless Chrome UA 访问极客时间，服务器返回 HTTP 451（法律限制不可用）。

**原因**：极客时间检测到 Headless UA 后直接拒绝请求。

**解决**：硬编码真实 Chrome UA 字符串（`Chrome/136.0.0.0`），所有 browser context 统一设置。

### 5.2 Cookie 过期 — acw_tc 30 分钟失效

**现象**：保存的 `storageState` 中包含 `acw_tc` cookie，30 分钟后过期。再次使用旧值请求时服务器返回 451。

**原因**：`acw_tc` 是极客时间的反爬 token，有效期仅 30 分钟。旧值不仅无效，还会导致服务器直接拒绝请求。

**解决**：加载 `storageState` 时主动过滤掉 `acw_tc` cookie，让服务器在首次请求时重新下发新 token。同时过滤已过期的 cookie（`expires < now`）。

### 5.3 登录按钮是 div 而非 button

**现象**：极客时间登录页的"登录"按钮是 `<div>` 元素，不是 `<button>`，无法通过常规的 `button` 选择器点击。

**解决**：遍历 DOM 树，找到 `textContent === "登录"` 且没有子元素的叶子节点（即按钮文字本身所在的元素），对其执行 `.click()`。

### 5.4 登录页默认是短信验证模式

**现象**：打开登录页默认显示短信验证码登录，需要切换到密码登录。

**解决**：定位 `a:has-text("密码登录")` 链接并点击，切换到密码输入表单。

### 5.5 SPA 内容异步加载

**现象**：页面 DOM 加载完成时（`domcontentloaded`）正文容器已存在但内容为空，需要等待 AJAX 请求完成后内容才填充。

**解决**：不使用 `waitForSelector`（它只等 DOM 出现），而是使用 `waitForFunction` 轮询正文容器内的文本长度超过 200 字符，确保 AJAX 内容已实际加载。

### 5.6 图片懒加载

**现象**：正文中的图片使用懒加载，只有滚动到可视区域时才加载。直接导出 PDF 会得到图片占位符。

**解决**：自动滚动页面到底部再回顶部（分 20 步，每步 1200px，间隔 100ms），触发所有懒加载图片的加载请求。

### 5.7 空白 PDF

**现象**：部分文章因加载失败或登录态过期，导出的 PDF 内容为空白（只有排版框架没有正文）。

**解决**：导出后校验 PDF 文件体积，小于 50KB 视为空白，自动删除并重试一次。

## 六、后续优化方向

### 6.1 短期优化

| 方向 | 说明 |
|------|------|
| **Chrome 路径自适应** | 当前硬编码 macOS Chrome 路径，可自动检测 Linux/Windows 下的 Chrome 路径，或通过环境变量配置 |
| **导出元数据** | PDF 文件属性中嵌入文章标题、作者、课程名等信息（Playwright PDF 支持 metadata） |
| **目录书签** | 生成包含课程目录书签的合并 PDF，方便阅读导航 |
| **进度持久化** | 将导出进度写入文件（如 JSON），支持 Ctrl+C 后精确续导，而非依赖文件名前缀匹配 |

### 6.2 中期优化

| 方向 | 说明 |
|------|------|
| **Cookie 自动刷新** | 监控 `acw_tc` 过期时间，在接近过期时主动刷新，避免导出过程中途失效 |
| **智能重试策略** | 区分失败原因（登录过期 vs 网络超时 vs 内容加载失败），针对性重试 |
| **并发度可配置** | 当前固定 3 并发，可改为 CLI 参数 `--concurrency N`，适应不同网络环境 |
| **内容完整性校验** | 不仅校验文件体积，还检查 PDF 页数（至少 > 1 页），防止导出只有封面没有正文的情况 |

### 6.3 长期优化

| 方向 | 说明 |
|------|------|
| **选择器自适应** | 极客时间改版可能导致选择器失效，可引入多种选择器策略 + 自动降级 + 告警机制 |
| **支持多平台** | 扩展支持其他内容平台（如掘金小册、知乎专栏），抽象出通用的"登录态管理 + 内容提取 + PDF 生成"框架 |
| **Docker 化部署** | 封装为 Docker 镜像，内置 Chrome，支持 CI/CD 定时导出（如每周自动导出新增内容） |
| **合并为电子书** | 将多篇 PDF 合并为完整的 EPUB/MOBI 电子书格式，支持 Kindle 等阅读器 |

---

## 七、维护要点

1. **UA 字符串**：Chrome 版本更新后需同步更新 UA，否则可能被检测
2. **DOM 选择器**：极客时间前端改版会导致正文容器选择器失效，需定期验证
3. **登录流程**：极客时间登录页改版（如增加验证码、改版 UI）需同步更新登录逻辑
4. **API 接口**：`/serv/v1/column/articles` 接口参数或返回格式变更需适配
5. **Playwright 版本**：Playwright 大版本升级可能有 breaking change，需验证兼容性
